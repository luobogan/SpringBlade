## 一、概述

### 1.1 什么是表单建模

表单建模是泛微 e-cology OA 系统的**核心低代码/零代码开发平台**，允许**非技术用户**（业务人员、管理员）通过**可视化配置界面**自定义业务表单、字段、流程和数据管理方式，而无需编写代码。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| **可视化表单设计** | 拖拽式配置表单字段、布局、校验规则 |
| **字段类型丰富** | 单行文本、多行文本、数字、日期、下拉框、复选框、单选按钮、浏览框、附件、HTML 编辑器等 |
| **流程触发** | 表单数据满足条件时自动触发工作流审批 |
| **明细表支持** | 一对多子表数据管理 |
| **数据导入导出** | Excel/XML 批量导入导出 |
| **权限控制** | 字段级、数据级、操作级权限 |
| **自定义查询** | 灵活的条件查询和视图配置 |
| **树形结构** | 模块数据支持树形分类 |
| **移动端适配** | 自动适配移动端显示 |
| **扩展接口** | Java 扩展代码、ESB 集成、WebService 接口 |

## 四、核心业务流程

### 4.1 数据保存流程（`ModeDataManager.saveModeData()`）

```
用户提交表单数据
        │
        ▼
  ModeDataManager.setRequest(request)  // 设置请求参数
        │
        ▼
  ModeDataManager.saveModeData()       // 核心入口
        │
        ├── src="del"? ──► 删除主表及明细数据
        │
        ├── iscreate="1"? ──► 通过 ModeDataIdUpdate 生成新 ID
        │
        ├── 遍历 workflow_billfield 字段表
        │    ├── 根据 HTML 类型处理字段值（文本/数字/附件/HTML等）
        │    ├── 处理 CLOB 长字段
        │    └── 组装 SQL（INSERT/UPDATE）
        │
        ├── 执行 SQL 操作主表
        │
        ├── 处理明细表数据（多明细表支持）
        │
        ├── 触发提醒功能 (TaskService)
        │
        ├── 判断是否触发工作流审批 (ModeDataApproval)
        │
        └── 记录操作日志 (FormmodeLog)
```

### 4.2 表单渲染流程

```
用户请求表单页面
        │
        ▼
  FormmodeFormAction / FormmodeFormController 接收请求
        │
        ▼
  读取模块配置（表单ID、字段定义、布局）
        │
        ▼
  读取 workflow_bill / workflow_billfield 表
        │
        ▼
  解析字段 HTML 类型（ElementPluginService）
        │
        ▼
  ResolveFormMode 将配置解析为 HTML 页面
        │
        ▼
  返回渲染后的表单页面给前端
```

### 4.3 流程触发流程

```
数据保存完成
        │
        ▼
  ModeDataApproval 判断触发条件
        │
        ├── 检查触发设置 (triggerWorkflowSetId)
        │    ├── 触发操作类型（新建/编辑/删除）
        │    └── 显示条件 (GetShowCondition)
        │
        ├── 字段映射：流程字段 ←→ 模块字段
        │
        ├── 创建工作流实例 (WorkflowToMode)
        │
        ├── 等待审批完成
        │
        └── 回写审批结果到模块数据
             ├── 成功回写 (successwriteback)
             └── 失败回写 (failwriteback)
```

## 十一、技术亮点

### 11.1 动态表结构

- 表单字段定义存储在 `workflow_billfield` 表
- 实际数据存储在动态创建的 `formtable_main_{billid}` 表中
- 字段的增加/删除不会丢失已有数据
- 支持字段历史版本管理

### 11.2 缓存机制

- **`ModeCacheManager`**（54 KB）：模板缓存管理器，缓存模块配置以减少数据库查询
- **`FormmodeRedisUtil`**：Redis 分布式缓存支持
- **`ResetFormmodeBrowserCache`**：浏览框缓存重置

### 11.3 大文件处理

- 导入导出支持 Excel/XML 格式
- `FormmodeData` 导出类 56 KB，导入类 152 KB
- `exports/generates/FormmodeXmlFactory.java` — XML 生成工厂
- `imports/parses/FormmodeXml.java` — XML 解析器

### 11.4 安全控制

- 权限精细化：可见、操作、字段、数据四级权限
- 字段加密：`ModeFormFieldEncryptComInfo`
- Token 验证：`FormModeConfig.istoken`
- 操作日志：`FormmodeLog` 日志记录
