# SpringBlade Cloud 项目规则文档

## 1. 项目概述

SpringBlade Cloud 是基于 Spring Cloud 的微服务架构版本，提供完整的微服务开发、部署和管理解决方案。本规则文档旨在规范 SpringBlade Cloud 项目的开发流程、技术规范和最佳实践。

## 2. 环境准备

### 2.1 开发环境
- **JDK**: 17+
- **Maven**: 3.8+
- **Node.js**: 18+ (前端开发)
- **IDE**: IntelliJ IDEA 或 Eclipse
- **Git**: 2.0+

### 2.2 基础设施
- **Nacos Server**: 3.1.1+ (服务注册与配置中心)
- **Redis Server**: 7.0+ (缓存)
- **MySQL**: 8.0+ (数据库)
- **Sentinel**: 2.0+ (服务保护)
- **Seata**: 2.0+ (分布式事务)

## 5. 微服务架构规范

### 5.1 服务设计原则
- **单一职责**: 每个服务只负责一个业务领域
- **服务自治**: 服务独立部署、独立扩展
- **数据隔离**: 每个服务有自己的数据库
- **API 契约**: 服务间通过 API 进行通信
- **容错设计**: 服务需要具备容错能力

### 5.2 服务间通信
- **同步通信**: 使用 OpenFeign 进行 REST 调用
- **异步通信**: 使用 RocketMQ 进行消息传递
- **通信规范**: 
  - 使用 HTTP/HTTPS 协议
  - 响应格式统一使用 `R` 包装类
  - 错误处理使用统一的异常体系

### 5.3 网关配置
- **路由规则**: 基于服务名的动态路由
- **过滤器**: 认证、日志、限流等
- **跨域配置**: 支持 CORS
- **安全配置**: 防止恶意请求

### 5.4 配置中心
- **配置管理**: 使用 Nacos 管理配置
- **配置结构**: 按环境、服务分层
- **配置加密**: 敏感配置加密存储
- **配置刷新**: 支持动态配置刷新

## 6. 命名规范

### 6.1 包命名
- **格式**: `org.springblade.{模块名}.{功能模块}`
- **示例**: `org.springblade.auth.controller`

### 6.2 类命名
- **控制器**: `{业务名称}Controller.java`
- **服务接口**: `I{业务名称}Service.java`
- **服务实现**: `{业务名称}ServiceImpl.java`
- **实体类**: `{业务名称}.java`
- **数据传输对象**: `{业务名称}DTO.java`
- **视图对象**: `{业务名称}VO.java`
- **映射器**: `{业务名称}Mapper.java`
- **工具类**: `{功能}Util.java`

### 6.3 方法命名
- **查询方法**: `get{业务名称}`、`find{业务名称}`、`list{业务名称}`
- **新增方法**: `save{业务名称}`、`add{业务名称}`
- **更新方法**: `update{业务名称}`
- **删除方法**: `remove{业务名称}`、`delete{业务名称}`
- **批量操作**: `batch{操作}{业务名称}`

### 6.4 数据库命名
- **表名**: `blade_{模块名}_{表名}`
- **字段名**: 下划线分隔

### 6.5 API 端点命名
- **格式**: `/api/{模块名}/{资源}/{操作}`
- **HTTP 方法**: GET (查询)、POST (新增)、PUT (更新)、DELETE (删除)

## 7. 代码规范

### 7.1 代码风格
- 使用 4 个空格进行缩进
- 每行不超过 120 个字符
- 大括号使用 K&R 风格
- 方法之间空一行
- 逻辑块之间空一行

### 7.2 注释规范
- **类注释**: 包含类的功能描述、作者、创建日期
- **方法注释**: 包含方法功能、参数、返回值
- **字段注释**: 包含字段含义
- **TODO 注释**: 标记待完成的功能

### 7.3 异常处理
- 使用 `ServiceException` 处理业务异常
- 使用 `GlobalExceptionHandler` 统一处理异常
- 异常信息应该清晰明确

### 7.4 日志规范
- 使用 `@Slf4j` 注解
- 日志级别使用规范: `debug`、`info`、`warn`、`error`
- 日志内容应该包含必要的上下文信息

## 8. 安全规范

### 8.1 认证授权
- 使用 JWT 进行身份认证
- 使用 Spring Security 进行权限控制
- 实现细粒度的权限管理

### 8.2 数据安全
- 敏感数据加密存储
- 防止 SQL 注入
- 防止 XSS 攻击
- 防止 CSRF 攻击

### 8.3 网络安全
- 使用 HTTPS
- 合理配置 CORS
- 防止 DDoS 攻击

## 9. 性能优化

### 9.1 数据库优化
- 合理设计索引
- 优化 SQL 查询
- 使用连接池
- 批量操作减少数据库交互

### 9.2 缓存策略
- 合理使用 Redis 缓存
- 设置适当的缓存过期时间
- 实现缓存一致性

### 9.3 服务优化
- 使用异步处理
- 合理设置线程池
- 优化序列化方式
- 减少网络传输

## 12. 版本管理

### 12.1 版本号规范
- **格式**: `{主版本}.{次版本}.{修订版本}`
- **示例**: `4.9.0`

### 12.2 分支管理
- **master**: 主分支，稳定版本
- **develop**: 开发分支
- **feature/{功能}**: 功能分支
- **bugfix/{问题}**: 修复分支

### 12.3 提交规范
- **格式**: `{类型}({范围}): {简短描述}`
- **类型**: feat, fix, docs, style, refactor, test, chore
- **示例**: `feat(auth): 添加社交登录功能`
